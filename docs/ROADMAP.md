## Phase 5: Claude Monitoring Dashboard (상시 모니터링)

### 목적

Claude Code 사용 현황을 **상시 백그라운드로 모니터링**하고, 스킬/플러그인의 실제 사용 빈도와 효과성을 한눈에 파악할 수 있는 대시보드.

### 해결하려는 문제

- 설치된 스킬/플러그인이 많지만, 실제로 어떤 것이 자주 쓰이는지 알 수 없음
- Hook이 제대로 동작하는지 실시간으로 확인할 방법이 없음
- 세션별 토큰 사용량, 도구 호출 빈도 등 비용/효율 관련 인사이트가 없음
- MCP 서버 상태(연결/오류)를 수동으로 확인해야 함

### 핵심 기능

#### 1. 상시 백그라운드 실행

- `claude-hub monitor` 명령으로 데몬 프로세스 시작
- `~/.claude/` 디렉토리를 watchdog(파일 감시)으로 모니터링
- 세션 로그(`.jsonl`)를 실시간 파싱
- 시스템 트레이 또는 메뉴바 아이콘으로 상태 표시 (macOS)
- 웹 대시보드는 `localhost:3847/monitor`에서 항상 접근 가능

#### 2. 스킬/플러그인 사용 현황

| 지표 | 데이터 소스 | 설명 |
|------|------------|------|
| 호출 횟수 | 세션 JSONL 로그 | 각 스킬/플러그인이 몇 번 호출되었는지 |
| 최근 사용일 | 세션 JSONL 로그 | 마지막으로 호출된 시점 |
| 미사용 스킬 | 호출 횟수 0 | 설치만 되고 한 번도 안 쓴 스킬 |
| 프로젝트별 사용 | 프로젝트 세션 매핑 | 어떤 프로젝트에서 어떤 스킬을 쓰는지 |

**UI**: 스킬별 히트맵 + 시간대별 사용 트렌드 차트

#### 3. Hook 실행 모니터링

| 지표 | 설명 |
|------|------|
| 실행 횟수 | 각 Hook 이벤트별 트리거 카운트 |
| 성공/실패 비율 | Hook 명령어 exit code 기반 |
| 실행 시간 | 각 Hook 실행에 걸린 ms |
| 타임아웃 발생 | 설정된 timeout을 초과한 횟수 |

**UI**: Hook 이벤트 로그 타임라인 + 에러 알림

#### 4. MCP 서버 상태

| 지표 | 설명 |
|------|------|
| 연결 상태 | UP/DOWN/ERROR |
| 응답 시간 | 평균 latency |
| 에러 로그 | 최근 에러 메시지 |

**UI**: 서버별 상태 카드 + health check ping

#### 5. 세션 통계

| 지표 | 데이터 소스 |
|------|------------|
| 일별 세션 수 | `stats-cache.json` + 세션 JSONL |
| 세션당 평균 메시지 수 | JSONL 파싱 |
| 도구 호출 분포 | JSONL의 tool_use 이벤트 |
| 프로젝트별 활동량 | 프로젝트 디렉토리별 세션 카운트 |

**UI**: 일별/주별 활동 차트 + 도구 사용 파이차트

#### 6. 효과성 인사이트

- **"죽은 스킬" 알림**: 30일 이상 미사용 스킬 리스트
- **자주 쓰는 조합**: 함께 사용되는 스킬/플러그인 패턴
- **비용 추정**: 세션당 예상 토큰 사용량 (메시지 길이 기반 추정)
- **추천**: 사용 패턴 기반으로 마켓플레이스에서 유사 스킬 추천

### 기술 설계 (초안)

```
claude-hub monitor
    │
    ├── Watchdog (파일 감시)
    │   ├── ~/.claude/sessions/ 변경 감지
    │   ├── ~/.claude/projects/*/**.jsonl 실시간 파싱
    │   └── ~/.claude/stats-cache.json 변경 감지
    │
    ├── Aggregator (집계)
    │   ├── 스킬/플러그인별 호출 카운트
    │   ├── Hook 실행 로그 수집
    │   ├── 시간대별 통계 집계
    │   └── SQLite에 저장 (~/.claude-hub/monitor.db)
    │
    └── Dashboard (웹 UI)
        ├── /monitor — 메인 모니터링 대시보드
        ├── /monitor/skills — 스킬별 상세 사용 현황
        ├── /monitor/hooks — Hook 실행 로그
        ├── /monitor/sessions — 세션 통계
        └── WebSocket — 실시간 업데이트
```

**데이터 저장**: SQLite (`~/.claude-hub/monitor.db`)
- 가벼움, 서버리스, 단일 파일
- 시계열 데이터 집계에 충분
- 파이썬 표준 라이브러리 sqlite3 사용

**실시간 업데이트**: WebSocket
- 파일 변경 감지 → WebSocket push → 브라우저 자동 갱신

### 의존성 추가

```
watchdog    # 파일 시스템 감시
aiosqlite   # 비동기 SQLite
```

### 우선순위

1. JSONL 세션 로그 파서 구현
2. SQLite 스키마 + Aggregator
3. 스킬/플러그인 사용 현황 API + UI
4. Hook 모니터링
5. 세션 통계
6. MCP 서버 상태
7. 효과성 인사이트 (추천 등)

## 향후 아이디어 (미정)

> 아래는 아직 구체화되지 않은 아이디어. 필요 시 별도 Phase로 구체화.

- **스킬 A/B 테스트**: 같은 작업을 다른 스킬 조합으로 실행하고 결과 비교
- **팀 공유 설정**: 팀원들의 claude-hub 설정을 Git repo로 공유/동기화
- **스킬 작성 도우미**: UI에서 SKILL.md를 시각적으로 작성하는 위자드
- **프롬프트 라이브러리**: 자주 쓰는 프롬프트를 저장/재사용
- **비용 대시보드**: Anthropic API 사용량 연동 (API key 필요)
