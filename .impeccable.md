## Design Context

### Users
Claude Code를 사용하는 개발자. 터미널 환경에 익숙하고, 효율성을 중시하며, 설정과 도구를 정밀하게 제어하고 싶어하는 엔지니어. 주로 로컬 개발 환경(macOS/Linux)에서 사용.

### Brand Personality
**Professional, Precise, Engineered**
- 전문 도구를 다루는 엔지니어의 느낌
- 불필요한 장식 없이 정보 밀도 높음
- 신뢰감 있고 안정적인 인상

### Aesthetic Direction
**Developer Terminal** — IDE/터미널 감성의 컴팩트하고 정보 밀도 높은 인터페이스

- **Theme**: Dark only + Emerald/Teal accent
- **Color Palette**:
  - Background: 깊은 다크 (zinc-950 기반, 약간 녹색 틴트)
  - Surface: zinc-900/zinc-800
  - Primary accent: emerald-400 (#34d399)
  - Secondary accent: teal-400 (#2dd4bf)
  - Success: emerald
  - Warning: amber
  - Danger: red-400
  - Text primary: zinc-100
  - Text secondary: zinc-400
  - Text muted: zinc-500
- **Typography**: JetBrains Mono (코드/숫자), Inter (UI 텍스트) — 모노스페이스 기반 터미널 느낌
- **Layout**: 컴팩트, 높은 정보 밀도, GitHub/Linear 스타일의 테이블과 리스트
- **Icons**: Lucide React — 가볍고 일관된 선형 아이콘
- **Anti-references**: 글래스모피즘, 네온 글로우, 그라디언트 텍스트, 바운스 애니메이션
- **References**: GitHub Dark, Linear, Vercel Dashboard, Warp Terminal

### Design Principles
1. **Information Density** — 스크롤 없이 최대한 많은 정보를 보여준다. 카드보다 테이블, 여백보다 콘텐츠.
2. **Terminal Aesthetic** — 모노스페이스 타이포, 커맨드 팔레트 스타일, 키보드 숏컷 힌트.
3. **Functional Motion** — 애니메이션은 상태 변화 피드백 용도로만. 장식적 모션 금지.
4. **Precision Over Polish** — 반짝이는 효과보다 정확한 정보 전달. 엔지니어가 신뢰할 수 있는 도구.
5. **Consistent Visual Language** — emerald accent은 "활성/성공"을 의미, 모든 곳에서 동일하게.
